# Deep Codebase Reader

> 系统化精读任意 GitHub 开源项目，生成完整的 `explain/` 文档体系。

## 用途

让初学者不需要逐行读源码就能吃透一个陌生项目。

## 触发场景

- "带我读懂这个项目"
- "精读这个代码库"
- "帮我理解这个项目的架构"
- "生成 explain 文档"

## 输出物

一套完整的 `explain/` 文档：

```
explain/
├── README.md                 # 导航地图
├── 00_阅读指南.md            # 如何使用这份文档
├── 01_项目总览.md            # 项目定位、技术栈、架构图
├── 02_程序执行流程.md        # 从启动到处理请求的完整流程
├── 03_模块与文件详解.md      # 所有模块索引
├── 04_函数和类详解.md        # 核心函数/类详解
├── 05_核心数据流.md          # 数据在各层之间如何流动
├── 06_配置_环境与依赖.md     # 环境变量、配置文件、依赖说明
├── 07_初学者易错点与难点.md  # ≥10 条易错点
├── 08_如何修改这个项目.md    # 常见扩展操作指南
├── 09_术语表.md              # 项目特有术语
├── 10_总复盘.md              # 六步检查表
├── modules/                  # 模块详解
├── functions/                # 函数详解
└── flows/                    # 关键流程详解
```

## 核心原则

- **面向初学者**，不是面向工程师
- 每份文档都回答"为什么"，不只是"是什么"
- 读完 `explain/` 之后，读者应该能自己去读源码

## 使用方式

将 `SKILL.md` 配置到你的 AI Agent（如 OpenCode、Claude）的 skills 目录中。
